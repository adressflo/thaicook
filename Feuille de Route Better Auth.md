Feuille de Route d'Intégration Better AuthCe document détaille les étapes pour migrer l'authentification de Firebase Auth vers Better Auth dans l'application Chanthana Thaï Cook, en utilisant Prisma et Supabase comme base de données, et en suivant une structure avec des pages dédiées pour l'inscription et la connexion.Objectif : Remplacer complètement Firebase Auth par Better Auth pour une solution d'authentification moderne, type-safe et auto-hébergée, tout en conservant la logique métier et les rôles (client/admin) existants.Phase 1 : Configuration Initiale (Terminée ✅)[x] Installer Better Auth : Ajout de la dépendance better-auth au projet via npm.[x] Configurer les Variables d'Environnement : Ajout de DATABASE_URL (Supabase via Prisma), BETTER_AUTH_URL, et BETTER_AUTH_SECRET dans .env.local.[x] Préparer la Base de Données (Prisma + Supabase) :[x] Ajout des modèles User, Account, Session, Verification au fichier prisma/schema.prisma (via npx @better-auth/cli generate et corrections manuelles).[x] Application des changements à la base de données Supabase via npx prisma db push.[x] Créer les Fichiers de Configuration Better Auth :[x] lib/auth.ts : Configuration de l'instance betterAuth avec prismaAdapter.[x] app/api/auth/[...all]/route.ts : Création du gestionnaire d'API (Mount Handler).[x] lib/auth-client.ts : Création du client React pour l'interaction frontend.[x] Relier Auth User au Profil client_db :[x] Modification de prisma/schema.prisma pour ajouter auth_user_id et la relation @relation dans client_db.[x] Ajout de la relation inverse client_profile dans User.[x] Application de la migration via npx prisma migrate dev.Phase 2 : Implémentation de l'Authentification de Base (En cours ⏳)[ ] 1. Créer la Page d'Inscription (app/auth/signup/page.tsx) :Créer une nouvelle page dédiée à /auth/signup.Design : S'inspirer fortement du design (logo, couleurs, disposition, composants Shadcn UI) de l'ancienne page app/profil/page.tsx [cite:uploaded:page.tsx] lorsqu'un utilisateur n'était pas connecté.Formulaire : Implémenter le formulaire en utilisant react-hook-form et zod.Champs :Inclure les champs obligatoires : email, password, nom, prenom, numero_de_telephone. La validation Zod doit les marquer comme requis.Inclure tous les autres champs présents dans le formulaire de app/profil/page.tsx (adresse, préférences, etc.) mais les rendre optionnels dans le schéma Zod et dans le formulaire lui-même.Logique onSubmit :Appeler signUp.email (depuis lib/auth-client.ts) pour créer l'entrée dans la table User.Si succès : Appeler la Server Action createClientProfileAction (fichier actions/clientActions.ts) pour créer l'entrée correspondante dans client_db, en passant toutes les données du formulaire (obligatoires et optionnelles remplies) et en assignant role: 'client'.Afficher les messages de succès/erreur avec toast.Rediriger vers /auth/signin après une inscription réussie.Créer la Server Action actions/clientActions.ts : Contenir la fonction createClientProfileAction qui interagit avec Prisma pour créer l'entrée client_db liée à l'utilisateur User.[ ] 2. Créer la Page de Connexion (app/auth/signin/page.tsx) :Créer une nouvelle page dédiée à /auth/signin.Design : Reproduire visuellement la section de connexion de l'ancienne page app/profil/page.tsx [cite:uploaded:page.tsx] (logo, couleurs, disposition).Formulaire : Implémenter le formulaire (Email, Mot de passe) avec react-hook-form et zod.Logique onSubmit :Appeler signIn.email (depuis lib/auth-client.ts).Gérer callbackUrl pour la redirection.Appeler router.refresh() après succès.Afficher les messages de succès/erreur avec toast.Liens : Inclure un lien vers /auth/signup ("Pas encore de compte ?") et prévoir l'emplacement pour le lien "Mot de passe oublié".Connexions Sociales (Optionnel - Phase 4) : Prévoir l'emplacement pour les boutons "Continuer avec Google", etc.[ ] 3.
